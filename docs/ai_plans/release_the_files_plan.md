# Release The Files Feature Plan

## Overview
The `/garryreleasethefiles` command allows users to submit grievances that trigger an AI agent to conduct biased "investigations" using SQL data queries and web research. The agent adopts the persona of a federal investigator/conspiracy theorist who treats every case with utmost seriousness while displaying extreme bias based on a random "for" or "against" determination.

## Core Requirements

### Command Structure
```
/garryreleasethefiles grievance:<string> [bribe:<integer>]
```
- `grievance`: Required text describing the user's complaint/concern
- `bribe`: Optional GarryCoin amount to influence case bias (positive bias)

### Database Schema Changes
New tables needed:

#### `release_files_cases`
```sql
CREATE TABLE release_files_cases (
    id SERIAL PRIMARY KEY,
    submitter_user_id VARCHAR(255) NOT NULL,
    grievance TEXT NOT NULL,
    bias_direction VARCHAR(10) NOT NULL, -- 'for' or 'against'
    bribe_amount INTEGER DEFAULT 0,
    case_status VARCHAR(20) DEFAULT 'active', -- 'active', 'closed'
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### `release_files_queries` 
```sql
CREATE TABLE release_files_queries (
    id SERIAL PRIMARY KEY,
    case_id INTEGER REFERENCES release_files_cases(id),
    query_text TEXT NOT NULL,
    query_results JSONB,
    interpretation TEXT,
    supports_bias BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### `release_files_codebase_refs`
```sql
CREATE TABLE release_files_codebase_refs (
    id SERIAL PRIMARY KEY,  
    case_id INTEGER REFERENCES release_files_cases(id),
    file_path TEXT NOT NULL,
    relevant_content TEXT,
    interpretation TEXT,
    supports_bias BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### Agent Persona & Behavior
The AI agent must:
1. **BOMBASTIC INVESTIGATOR CHARACTER**: ALL CAPS, Trump-style proclamations, "EXPLOSIVE FINDINGS", "TREMENDOUS CORRUPTION"
2. **Display Extreme Bias**: Interpret ALL data to support the randomly chosen position with ABSOLUTE CERTAINTY
3. **Never Break Character**: Treat every investigation as WORLD-CHANGING SCANDAL
4. **Be Consistent**: Users/topics should receive similar treatment - if someone was "CROOKED" before, they remain "TOTALLY CORRUPT"
5. **Call Out Other Users**: LOUDLY implicate Discord users - "SLEEPY JOE'S GAMBLING RECORDS ARE A DISASTER!"

### Technical Implementation

#### Phase 1: Core Infrastructure
1. **Database Migration**: Add the three new tables
2. **Command Registration**: Add `/garryreleasethefiles` to `command_definitions.js`
3. **Basic Command Handler**: Create initial response structure
4. **Bias Assignment**: Random 50/50 split, with bribe weighting (75/25 if bribed)

#### Phase 2: SQL Investigation Engine
1. **Query Generation**: Agent analyzes grievance and generates relevant SQL queries
2. **Safe Execution**: Use existing read-only database connection patterns
3. **Data Interpretation**: Bias all query results toward predetermined position
4. **Result Storage**: Save queries and interpretations to database

#### Phase 3: Codebase Documentation Integration
1. **File Analysis**: Agent can read project documentation files (CLAUDE.md, DESIGN.md, etc.)
2. **Code Reference**: Pull relevant snippets from source files to support narrative
3. **Technical Evidence**: Use codebase patterns as "PROOF" of conspiracy/innocence
4. **Documentation Storage**: Save codebase references and interpretations

#### Phase 4: Case Consistency
1. **User History Tracking**: Query previous cases involving mentioned users
2. **Topic Pattern Recognition**: Identify recurring themes and maintain consistent stance
3. **Cross-Reference System**: Link related cases and maintain narrative coherence

#### Phase 5: Response Generation
1. **Case File Assembly**: Compile all SQL queries, web sources, and interpretations
2. **Federal Report Format**: Professional investigation summary with classifications
3. **User Implications**: Call out positive/negative involvement of other Discord users
4. **Evidence Presentation**: Display "supporting" data points with authoritative tone

### Bribery System
- **No Bribe**: 50/50 random chance for "for" vs "against"
- **With Bribe**: 75% chance of "for" (favorable) outcome
- **Bribe Cost**: Deducted from user balance before investigation
- **Bribe Logging**: Record in transactions table as `release_files_bribe`

### Security Considerations
1. **SQL Injection Prevention**: All queries generated by AI, never direct user input
2. **Read-Only Access**: Use existing `readOnlyDb` patterns
3. **Rate Limiting**: One investigation per user per day (like loan limits)
4. **Content Filtering**: Sanitize grievance input for Discord TOS compliance

### Example Flow
1. User: `/garryreleasethefiles grievance:"Bob keeps winning at heist" bribe:100`
2. System: Deducts 100 GC, rolls bias (75% chance "for" user due to bribe)
3. Agent: Analyzes grievance, generates SQL queries about Bob's heist performance
4. Agent: Reads heist.js code file to find "SMOKING GUN EVIDENCE" in the algorithm
5. Agent: Interprets ALL data to support user's position with MAXIMUM BOMBAST
6. Response: "BREAKING: CROOKED BOB'S HEIST EMPIRE EXPOSED! RIGGED SYSTEM!"
7. Database: Stores case, queries, and codebase references for future consistency

### Discord Integration
- **Defer Response**: Use deferred reply for investigation processing time
- **Channel Announcement**: Post that an investigation has been initiated
- **Progress Updates**: Optional interim messages during long investigations
- **Final Report**: Comprehensive case file with supporting evidence
- **User Mentions**: Tag implicated users in final report

### Consistency Engine
The agent will:
- **Check Previous Cases**: Query `release_files_cases` for prior investigations of mentioned users
- **Maintain Character Arc**: Users who were "problematic" in past cases remain problematic
- **Topic Memory**: Remember stance on recurring issues (gambling, economy, etc.)
- **Escalating Narratives**: Build on previous investigations to create ongoing storylines

This feature will create an engaging meta-game where users can "investigate" each other while the AI maintains serious investigative roleplay with hidden bias mechanics.
